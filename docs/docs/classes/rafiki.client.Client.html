Creates a Rafiki advisor. If advisor_id is passed, it will create an advisor of that ID, or do nothing if an advisor of that ID has already been created.

Parameters:
  • knob_config (dict[str, any]) – Knob configuration for advisor session
  • advisor_id (str) – ID of advisor to create
create_inference_job(app, app_version=-1)[source]¶

Creates and starts running a inference job on Rafiki with the 2 best trials of an associated train job. The inference job is tagged with the train job’s app and app version.

In this method’s response, query_host is this inference job’s query front-end’s host.

Parameters:
  • app (str) – Name of the app identifying the train job to use
  • app_version (str) – Version of the app identifying the train job to use
create_model(name, task, model_inst, docker_image=None)[source]¶

Creates a model on Rafiki.

Parameters:
  • name (str) – Name of the model, must be unique on Rafiki
  • task (str) – Task associated with the model, the model must adhere to the specification of the task
  • model_inst (obj) – A Python object that implements rafiki.model.BaseModel
  • docker_image (str) – A custom docker image name that extends rafiki_model
create_train_job(app, task, train_dataset_uri, test_dataset_uri, budget_type='MODEL_TRIAL_COUNT', budget_amount=10)[source]¶

Creates and starts running a train job on Rafiki.

Parameters:
  • app (str) – Name of the app associated with the train job
  • task (str) – Task associated with the train job, the train job will train models associated with the task
  • train_dataset_uri (str) – URI of the train dataset in a format specified by the task
  • test_dataset_uri (str) – URI of the test (development) dataset in a format specified by the task
  • budget_type (rafiki.constants.BudgetType) – Type of budget for the train job
  • budget_amount (int) – Budget amount in units specific to the budget type

login(email, password)[source]¶

Creates a login session as a Rafiki user. You will have to be logged in to perform any actions.

App developers can create, list and stop train and inference jobs, as well as list models. Model developers can create and list models.

Parameters:
  • email (str) – User’s email
  • password (str) – User’s password
